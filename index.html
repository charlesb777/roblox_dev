The core concept

Git vs Rojo

They both sound like "syncing," but each talks to a different thing. This is the part everyone gets wrong.

🔗 Rojo

The live bridge into Studio. The Rojo plugin talks to a running rojo serve and pulls your .luau files straight into Explorer.

Talks to → Roblox Studio

🗂️ Git / GitHub

Version control for your text files. Records every change so you can undo, branch, and back up online.

Talks to → your files
"If I have Git, why doesn't Studio just read it?"
Because Git only stores text files — Studio has no idea Git exists. Rojo is the bridge that pushes your files into Studio. Git runs alongside purely to save and back up your work. When a build is good, you publish it yourself from Studio.
Before you start

The two things everyone hits

Nail these and the rest is easy. Miss them and Rojo looks "broken" when it isn't.

🔴 Version mismatch

The Rojo CLI (terminal) and the Rojo Studio plugin must be the exact same version (e.g. both 7.7.0-rc.1). Different versions → "protocol mismatch" and Connect fails.

🟡 Connect ≠ sync

Connecting only means Studio can talk to Rojo. Rojo only syncs files mapped in default.project.json. Files outside those folders are invisible to Studio.

Step 6 · the "I connected but see no files" fix

Verify the sync

Rojo only syncs what default.project.json maps. After connecting, open Explorer — you should see this exact tree from the starter:

Roblox Studio — Explorer
ServerScriptService
└─ Server
   ├─ Main ← src/server/Main.server.luau
   └─ CoinSpawner ← src/server/CoinSpawner.server.luau
ReplicatedStorage
└─ Shared
   └─ GameConfig ← src/shared/GameConfig.luau
StarterPlayer
└─ StarterPlayerScripts
   └─ Client
      └─ Main ← src/client/Main.client.luau
Live test: add print("Hello from Rojo") to Main.server.luau, save, press Play — it appears in Output. If it does, sync works.

File names decide what Studio makes

File nameBecomes in Studio
Main.server.luauScript — runs on the server
Main.client.luauLocalScript — runs on each player
GameConfig.luauModuleScript — shared code you require
init.luau (in a folder)turns that folder into a ModuleScript
A file at roblox/script.luau — not under a mapped folder, or missing the .server/.client suffix — will not sync. Move it into src/server, src/client, or src/shared.

First-time Git setup (once per project)

git init git add . git commit -m "First working Rojo sync" git branch -M main git remote add origin https://github.com/YOUR_USERNAME/roblox-cursor-starter.git git push -u origin main # optional: a safe branch for the kid to code on git checkout -b dev git push -u origin dev
🔒 Keep it safe: no API keys, no auto-publish, never use your .ROBLOSECURITY cookie. Publish manually via File → Publish to Roblox.
